# v2026.03.27 - Agent 模式默认开启 & Workspace 附件存储

## 变更概述

### 1. Agent 模式默认开启
- Agent 模式不再是一个开关，而是始终开启的默认行为
- 每个新 session 创建时自动带有 `agentConfig`，继承全局默认设置
- 旧 session（无 agentConfig）在加载时自动获得默认配置
- 输入区的 Agent 按钮改为打开设置面板入口，不再有开/关切换
- Files 按钮始终显示，不再受 agent 模式条件限制

### 2. 全局 Agent 设置（Settings → Agent 标签页）
- 在 SettingsModal 中新增 "Agent" 标签页
- 可配置全局默认：系统 Prompt、最大迭代次数、默认 Skills、默认 MCP 服务器
- 保存到后端数据库（`user_settings.agent_defaults` 列）

### 3. Session 级设置继承全局
- Session 的 AgentConfigPanel 默认继承全局设置
- 每个配置项旁显示 "from global" 标识（表示继承中）或 "reset" 按钮（表示已覆盖，可重置）
- 面板标题改为 "Session Settings"，移除了 "Enable Agent Mode" 开关

### 4. 附件存储到 Workspace（IndexedDB）
- 所有上传的附件（图片 + 文档）存储到 IndexedDB，通过 `workspaceService.writeFile()`
- `Attachment` 类型新增 `workspacePath` 字段，记录文件在 workspace 中的路径
- 上传文件时若无 session，自动创建新 session
- 发送消息时自动从 IndexedDB 解析 workspace 路径回 data URL
- localStorage 持久化时仅保存 workspacePath（不含大体积 base64），彻底解决 QuotaExceededError
- ChatArea 新增 `WorkspaceImage` 组件，异步从 IndexedDB 加载图片显示

## 修改文件清单

### 后端
- `src/db/migrate.ts` — 新增 `agent_defaults TEXT` 列迁移（MySQL）
- `src/db/migrate-postgres.ts` — CREATE TABLE 和 ALTER TABLE 新增 `agent_defaults`（PostgreSQL）
- `src/services/userService.ts` — `getUserSettings` / `updateUserSettings` 读写 `agent_defaults`

### 前端类型
- `client/types.ts` — `AgentConfig` 移除 `enabled`；新增 `AgentDefaults` 接口；`AppSettings` 新增 `agentDefaults`；`Attachment` 新增 `workspacePath`

### 前端组件
- `client/App.tsx` — 核心改动：
  - 导入 `AgentDefaults`
  - `createNewSession` 自动附带 agentConfig（继承全局默认）
  - 旧 session 加载时补充默认 agentConfig
  - `processFile` 重构：写入 IndexedDB、保留 workspacePath
  - `handleSendMessage` 发送前解析 workspace 路径为 data URL
  - 移除所有 `agentConfig?.enabled` 条件判断
  - Agent 按钮和 Files 按钮始终显示
  - AgentConfigPanel 传入 `globalDefaults` prop
- `client/components/AgentConfigPanel.tsx` — 完全重写：
  - 移除 enabled 开关
  - 新增 `globalDefaults` prop
  - 每个配置项显示继承/覆盖状态和 reset 按钮
  - 标题改为 "Session Settings"
- `client/components/SettingsModal.tsx` — 新增 Agent 标签页：
  - 全局系统 Prompt、最大迭代次数、Skills、MCP 服务器配置
  - 保存到后端
- `client/components/ChatArea.tsx` — 新增 `WorkspaceImage` 组件，支持从 IndexedDB 异步加载图片

### 5. 文档附件提取文本不再显示在用户消息中
- 之前：PDF/Word 等文档的提取文本会追加到用户可见消息中，导致消息气泡很长
- 现在：用户消息仅显示用户输入的文本，提取的文档内容仅在构建 API 请求时注入
- 分离了 `messageContent`（用户可见）和 `apiContent`（发送给模型）
- 重新生成消息时同样注入提取文本（`regenerateWithContent`）

## 向后兼容
- 已有 session 无 agentConfig 时自动获得默认配置
- 已有 session 的 agentConfig 中的 `enabled` 字段被忽略（类型已移除）
- 无 workspacePath 的旧附件继续使用 URL 或占位图标显示
