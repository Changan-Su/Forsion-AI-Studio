# v2026.03.27 - 粘贴上传附件 & 文件大小限制

## 变更内容

### 1. 支持粘贴上传附件
- 在聊天输入框（textarea）添加 `onPaste` 事件处理
- 支持从剪贴板粘贴图片（如截图）、PDF、Word、Text 等文件
- 粘贴文件后自动走 `processFile` 流程，与拖拽上传和文件选择器上传行为一致
- 仅当剪贴板包含文件时拦截粘贴事件；纯文本粘贴不受影响

### 2. 附件大小限制 10MB
- 在 `processFile` 入口处新增文件大小校验（`MAX_FILE_SIZE = 10MB`）
- 超过 10MB 的文件不会被上传，触发弹窗提示
- 弹窗显示文件名和实际大小，提示用户压缩后重试
- 弹窗适配所有主题（default / notion / monet）

## 实现方式

### 粘贴上传
- **文件**: `client/App.tsx`
- 新增 `handlePaste` 函数，遍历 `e.clipboardData.items`，筛选 `kind === 'file'` 且 MIME 类型在支持列表中的项
- 将 `handlePaste` 绑定到输入框的 `onPaste` 属性
- 支持的 MIME 类型与拖拽上传一致：`image/*`, `application/pdf`, `application/msword`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`, `text/plain`, `text/markdown`

### 文件大小限制
- **文件**: `client/App.tsx`
- 新增常量 `MAX_FILE_SIZE = 10 * 1024 * 1024`
- 新增状态 `fileSizeError: { name: string; size: string } | null`
- `processFile` 函数开头检查 `file.size > MAX_FILE_SIZE`，超限时设置 `fileSizeError` 并 return
- 弹窗使用 fixed overlay + backdrop-blur，z-index 70，点击外部或按钮均可关闭
- 所有上传路径（文件选择器、拖拽到输入区、拖拽到聊天区、粘贴）均受此限制保护，因为统一经过 `processFile`

### 修复: localStorage QuotaExceededError
- **问题**: 上传图片后，base64 数据随消息存入 `localStorage`，两张图片即可超出 ~5MB 配额限制，导致应用崩溃
- **修复文件**: `client/App.tsx`, `client/components/ChatArea.tsx`
- 在 sessions 持久化时，剥离所有 `data:` 开头的附件 URL（base64 图片/文档），仅保留元信息（类型、文件名、MIME 类型）
- 超长 `extractedText`（>10KB）截断保存
- 用 try-catch 包裹 `localStorage.setItem` 作为兜底
- ChatArea 中为缺失 URL 的图片附件显示占位图标，避免空白

## 影响范围
- 修改 `client/App.tsx`、`client/components/ChatArea.tsx`，无后端变更
- 不影响已有功能（拖拽上传、文件选择器上传、头像上传等）
- 历史消息中的图片附件在页面刷新后显示为占位图标（base64 数据不再持久化），当前会话内仍正常显示
