# Theme Fix & Drag Overlay Redesign — v2026.03.28

## 概述

修复了 Apple 和 Forsion1 主题没有正常应用的问题，以及附件拖拽上传的 overlay 丑陋、未适配主题的问题。

---

## 问题 1：Apple / Forsion1 主题显示为 Default

### 根本原因

`client/index.html` 中定义了完整的 CSS 工具类（`.apple-glass`、`.bg-apple-surface`、`.forsion1-glass`、`.bg-forsion1-surface`），但所有组件的条件分支只使用 `isNotion` / `isMonet` / else，Apple 和 Forsion1 始终落入 `else` 分支，与 Default 主题完全相同。

### 修复方案

在 `App.tsx`、`Sidebar.tsx`、`ChatArea.tsx` 中增加 `isApple` 和 `isForsion1` 标志，并在所有主要 UI 区域添加专属分支。

### App.tsx 修改点

- 新增 `isApple = themePreset === 'apple'` 和 `isForsion1 = themePreset === 'forsion1'` 标志
- `appBackground`：Apple → `bg-apple-surface`，Forsion1 → `bg-forsion1-surface`
- 装饰渐变 overlays：不再在 Apple/Forsion1 主题下显示（它们有自己的平面底色）
- **Header**：Apple → `apple-glass`，Forsion1 → `forsion1-glass`
- **Model 名称 / Tagline**：Apple → 蓝色，Forsion1 → 琥珀色
- **Model 下拉面板**：Apple/Forsion1 各自使用 glass 类 + 对应边框色
- **输入区域 wrapper**：Apple/Forsion1 使用对应 glass 类
- **Composer 容器**：Apple/Forsion1 使用 glass + 对应 focus ring 色
- **附件 chip**：Apple 使用白/蓝色，Forsion1 使用石色
- **Paperclip / Expand 按钮**：Apple 蓝色 / Forsion1 琥珀色
- **Send 按钮**：Apple → `bg-blue-500`，Forsion1 → `bg-amber-700`

### Sidebar.tsx 修改点

- Sidebar 外壳：Apple → `apple-glass`，Forsion1 → `forsion1-glass`
- Logo 图标：Apple 蓝色背景，Forsion1 琥珀色背景
- Logo 文字：Apple 蓝色，Forsion1 琥珀色
- New Chat 按钮：Apple `bg-blue-500`，Forsion1 `bg-amber-700`
- 会话列表激活行：Apple 蓝色 tint，Forsion1 琥珀色 tint
- 会话列表 hover：Apple 淡蓝 hover，Forsion1 淡琥珀 hover
- Footer 边框：各主题使用对应边框色
- 用户头像：Apple 蓝色，Forsion1 琥珀色

### ChatArea.tsx 修改点

- 空状态/消息列表背景：Apple → `bg-apple-surface`，Forsion1 → `bg-forsion1-surface`
- 空状态 Hero 图标：Apple `bg-blue-500`，Forsion1 `bg-amber-700`
- 空状态标题/正文/链接颜色：匹配主题色
- 用户消息气泡：Apple `bg-blue-500`，Forsion1 `bg-amber-700`
- AI 消息气泡：Apple `apple-glass`，Forsion1 `forsion1-glass`
- 用户头像、Model 头像：Apple 蓝色，Forsion1 琥珀色

## 问题 2：拖拽 Overlay 丑陋且未适配主题

### 根本原因

两处 overlay（`App.tsx` 的输入区全屏 overlay 和 `ChatArea.tsx` 的空状态 overlay）使用硬编码颜色，无主题分支，也没有流畅的入场/退场动画。

### 修复方案

#### App.tsx 输入区 overlay（全局覆盖层）

旧：`fixed inset-0 bg-white/80 dark:bg-black/80 backdrop-blur-sm`，居中图标，无动画。

新：
- 使用 `AnimatePresence` + `motion.div` 实现 fade in/out
- 外层：主题适配的半透明背景 + `backdrop-blur-xl`
- 内层：居中卡片，带 **虚线边框**（`border-2 border-dashed`）、圆角（`var(--radius-xl)`）、弹簧入场动画
- 主题色映射：Notion（灰色）、Monet（紫色）、Apple（蓝色）、Forsion1（琥珀色）、Default（forsion）
- 图标尺寸从 64 降到 48，移除激进的视觉元素

#### ChatArea.tsx 空状态 overlay

旧：`border-4 border-dashed border-forsion-400 rounded-xl m-4`，`animate-bounce`，硬编码 forsion 色。

新：
- 使用 `AnimatePresence` + `motion.div` 实现 fade in/out
- 移除 `animate-bounce`，改为 Framer Motion 弹簧入场
- `border-2` 替代 `border-4`（更精致）
- `rounded-[var(--radius-xl)]` 替代 `rounded-xl`（使用圆角 token）
- 主题色完整映射（Notion/Monet/Apple/Forsion1/Default）
- 子内容独立 spring 动画（delay 0.05s，更有层次感）
