# v2026.03.28 - API 代理 401 导致误退出登录

## 问题描述

当用户在 AI Studio 中使用聊天功能时，如果后端代理请求上游 AI API（如 SiliconFlow）失败并返回 HTTP 401（API Key 无效/过期），前端会误判为"用户会话过期"，清除 auth token 并强制退出登录。

**复现路径**：发送消息 → 后端代理请求上游 API → 上游返回 401 → 前端退出登录

**影响范围**：`proxyChatCompletions`、`proxyImageGeneration`、`proxyImageEdit` 三个代理方法。

## 根因分析

Bug 链条涉及前后端两侧：

### 1. 后端透传上游状态码

`src/routes/chat.ts` 中，当上游 AI API 返回非 200 响应时，后端直接将上游的 HTTP 状态码透传给前端：

```javascript
return res.status(response.status).json({ detail: errorDetail });
```

如果上游 API（如 SiliconFlow）因 API Key 失效返回 401，后端就会原样返回 401 给前端。这个 401 的语义是"服务端的 API Key 无效"，而非"客户端的用户会话无效"。

### 2. 前端对所有 401 执行退出登录

`client/services/backendService.ts` 的代理方法中，收到 401 后一律执行：

```javascript
if (res.status === 401) {
  clearAuth();                  // 清除 localStorage 中的 token
  throw new AuthRequiredError(); // 抛出认证错误
}
```

### 3. App.tsx 捕获 AuthRequiredError 并执行登出

```javascript
if (error?.code === 'AUTH_REQUIRED' || error?.name === 'AuthRequiredError') {
  clearAuthState('Session expired. Please sign in again.');
}
```

## 解决方案

### 后端修复（主要）

在 `src/routes/chat.ts` 的三个代理端点（`/chat/completions`、`/images/generations`、`/images/edits`）中，将上游返回的 401/403 映射为 502（Bad Gateway），因为这类错误本质上是服务端配置问题：

```javascript
const clientStatus = (response.status === 401 || response.status === 403)
  ? 502
  : response.status;
return res.status(clientStatus).json({ detail: errorDetail });
```

### 前端修复（防御性）

在 `client/services/backendService.ts` 的三个代理方法中移除 `clearAuth()` + `AuthRequiredError` 逻辑。这些端点上的 401 不应触发退出登录，因为用户的 JWT 已经在 `authMiddleware` 中验证通过——只有 JWT 有效的请求才会到达代理逻辑。

## 修改文件

| 文件 | 修改内容 |
|------|----------|
| `src/routes/chat.ts` | 3 处：上游 401/403 映射为 502 |
| `apps/Forsion-AI-Studio/client/services/backendService.ts` | 3 个代理方法：移除 clearAuth + AuthRequiredError |

## 参考

- 后端 `authMiddleware` 负责验证用户 JWT，在代理逻辑之前执行
- 前端其他需要用户认证的 API（如 `getCurrentUser`、`getSettings` 等）保留 401 → 退出登录的行为，因为那些端点的 401 确实代表会话过期
