**版本**：v1.0

## 一、用户主流程（整体旅程）

```mermaid
flowchart TD
    openApp[用户打开 AI Studio] --> checkAuth{已登录?}
    checkAuth -- 否 --> loginPage[跳转登录页]
    loginPage --> inputCreds[输入用户名 + 密码 + 邀请码]
    inputCreds --> authVerify{验证通过?}
    authVerify -- 否 --> loginError["提示错误信息(密码错误/邀请码无效)"]
    loginError --> inputCreds
    authVerify -- 是 --> checkFirstLogin{首次登录?}
    checkAuth -- 是 --> mainChat[进入主聊天界面]
    checkFirstLogin -- 是 --> onboarding[触发新用户引导流程]
    checkFirstLogin -- 否 --> mainChat
    onboarding --> mainChat

    mainChat --> userAction{用户操作}
    userAction --> sendMsg[发送消息对话]
    userAction --> switchModel[切换 AI 模型]
    userAction --> openSettings[打开 Settings]
    userAction --> manageSession[管理会话]

    sendMsg --> aiProcess[AI 处理消息]
    aiProcess --> checkSkills{是否需要调用 Skill?}
    checkSkills -- 否 --> streamReply[流式输出回复]
    checkSkills -- 是 --> skillExec[进入 Skill 执行流程]
    skillExec --> streamReply
    streamReply --> userAction

    openSettings --> settingsTabs{Settings Tab}
    settingsTabs --> skillsMarket[Skills 市场]
    settingsTabs --> accountSettings[账户设置]
    settingsTabs --> themeSettings[主题切换]
    settingsTabs --> authManage[数据授权管理]
    skillsMarket --> userAction
    accountSettings --> userAction
    themeSettings --> userAction
    authManage --> userAction

    switchModel --> mainChat
    manageSession --> mainChat
```

## 二、Skills 安装与作用域管理

```mermaid
flowchart TD
    openMarket[用户进入 Skills 市场] --> browseSkills[浏览 Skill 卡片列表]
    browseSkills --> filterAction{筛选操作}
    filterAction --> filterAll[全部]
    filterAction --> filterOfficial[仅官方]
    filterAction --> filterInstalled[已安装]
    filterAction --> searchSkill[搜索关键词]
    filterAll --> viewCard[查看 Skill 详情卡片]
    filterOfficial --> viewCard
    filterInstalled --> viewCard
    searchSkill --> viewCard

    viewCard --> installAction{操作选择}
    installAction --> installBtn["点击安装"]
    installAction --> alreadyInstalled[已安装 - 管理作用域]
    installAction --> createCustom[创建自定义 Skill]

    installBtn --> checkCrossApp{是跨应用 Skill?}
    checkCrossApp -- 否 --> installSuccess[安装成功 - 默认全局启用]
    checkCrossApp -- 是 --> needAuth["标记为'需授权' - 首次调用时触发授权"]
    needAuth --> installSuccess

    installSuccess --> scopeConfig[进入作用域配置]
    alreadyInstalled --> scopeConfig

    scopeConfig --> scopeChoice{选择作用域}
    scopeChoice --> scopeGlobal["全局启用(所有 Session)"]
    scopeChoice --> scopeSession["仅当前 Session"]
    scopeChoice --> scopeDisabled[已禁用]

    scopeGlobal --> saveScope[保存配置]
    scopeSession --> saveScope
    scopeDisabled --> saveScope
    saveScope --> doneMsg["配置生效"]

    createCustom --> fillForm["填写: 名称 + 描述 + Tools + 提示词"]
    fillForm --> saveCustom[保存到 IndexedDB]
    saveCustom --> installSuccess
```

## 三、Skill 执行可视化流程

```mermaid
flowchart TD
    userMsg[用户发送消息] --> aiAnalyze[AI 分析意图]
    aiAnalyze --> needSkill{需要调用 Skill?}
    needSkill -- 否 --> directReply[直接生成文本回复]
    needSkill -- 是 --> checkEnabled{Skill 已启用?}

    checkEnabled -- 否 --> suggestEnable["提示: 该 Skill 未启用, 是否前往开启?"]
    suggestEnable --> userDecide{用户选择}
    userDecide -- 开启 --> enableSkill[启用 Skill] --> callTool
    userDecide -- 忽略 --> directReply

    checkEnabled -- 是 --> checkCrossAuth{是跨应用 Skill 且未授权?}
    checkCrossAuth -- 是 --> authDialog[弹出授权确认弹窗]
    authDialog --> authResult{用户选择}
    authResult -- 允许 --> saveAuth[保存授权] --> callTool
    authResult -- 拒绝 --> authDenied["AI 回复: 需要授权才能访问"]

    checkCrossAuth -- 否 --> callTool

    callTool["调用 Tool (显示'执行中'状态)"]
    callTool --> toolResult{执行结果}
    toolResult -- 成功 --> showSuccess["显示: 工具名 + 耗时 + 成功图标"]
    toolResult -- 失败 --> showFail["显示: 工具名 + 错误原因 + 重试按钮"]
    toolResult -- 超时 --> showTimeout["显示: 响应超时 + 重试/切换模型"]

    showSuccess --> expandable["可展开查看详细返回数据"]
    expandable --> aiCombine[AI 结合 Tool 结果生成回复]
    aiCombine --> streamReply[流式输出最终回复]

    showFail --> retryAction{用户操作}
    retryAction -- 重试 --> callTool
    retryAction -- 忽略 --> directReply

    showTimeout --> retryAction
```

## 四、跨应用数据授权流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant AI as AI Agent
    participant FE as 前端
    participant BE as 后端数据中枢
    participant APP as 目标 App(如 Calendar)

    U->>AI: "明天有什么重要会议?"
    AI->>AI: 意图识别 - 需要 Calendar 数据
    AI->>FE: 调用 Calendar Skill.get_events

    FE->>FE: 检查授权状态
    alt 未授权
        FE->>U: 弹出授权确认弹窗
        Note over U,FE: 展示: App 名称 + 权限明细 + 隐私说明
        alt 用户允许
            U->>FE: 点击"允许访问"
            FE->>BE: 保存授权记录(user_id + app_id + scope)
            BE-->>FE: 授权成功
        else 用户拒绝
            U->>FE: 点击"拒绝"
            FE->>AI: 授权被拒
            AI->>U: "需要访问 Calendar 才能回答, 可在 Settings 中授权"
        end
    end

    FE->>BE: 请求 Calendar 数据(带 user_id + auth_token)
    BE->>BE: 验证授权 + 查询用户数据分区
    BE->>APP: 读取 app_id=calendar 的数据
    APP-->>BE: 返回日程列表
    BE-->>FE: 返回结构化数据
    FE->>AI: Tool 执行结果
    AI->>U: "明天有 1 个重要会议: Q1 Review 10:00 AM..."
```

## 六、积分消耗与模型调用流程

```mermaid
flowchart TD
    userSend[用户发送消息] --> checkCredits{积分余额充足?}
    checkCredits -- 否 --> noCredits["提示: 积分不足"]
    noCredits --> creditAction{用户选择}
    creditAction --> viewBalance[查看余额明细]
    creditAction --> contactAdmin[联系管理员]

    checkCredits -- 是 --> buildReq[构建 API 请求]
    buildReq --> attachSkills[注入已启用 Skills 的 System Prompt]
    attachSkills --> callModel["调用所选模型 API(SSE)"]

    callModel --> modelResult{响应状态}
    modelResult -- 正常 --> streamTokens[逐 Token 流式输出]
    streamTokens --> calcCost[计算 Token 消耗 - 扣减积分]
    calcCost --> logUsage["记录使用日志(model + tokens + project_source)"]

    modelResult -- API Key 失效 --> keyError["提示: API 配置过期, 请切换模型"]
    modelResult -- 超时 --> timeout["30s 超时提示 + 重试/切换模型"]
    modelResult -- 限流 --> rateLimit["提示: 请求过于频繁, 请稍后重试"]
```
