一、需求背景

1.1 现状问题

问题一：AI 工具碎片化严重。用户需要在 ChatGPT、Claude、Gemini 等多个平台间反复切换，每个平台有独立的账号体系和对话历史，无法统一管理。非技术用户面临认知负担大、学习成本高的困境。

问题二：AI 能力止步于"对话"。主流 AI 工具以纯文本对话为主，缺乏可扩展的"行动能力"。用户无法让 AI 主动调用外部工具（如查日历、发邮件、搜索文件），只能手动复制粘贴信息。

问题三：应用间数据孤岛。用户在不同应用（日历、桌面、笔记）中产生的数据彼此隔离，AI 无法获得完整的用户上下文。例如日历中有明天的会议，但聊天 AI 无法主动提醒用户准备材料。

问题四：现有方案不支持自部署和自定义。企业和个人用户对数据隐私有顾虑，希望使用自有 API Key、部署在自己的服务器上，但现有 SaaS 产品无法满足这一需求。

1.2 为什么现在做？

1) 技术窗口期成熟：多家 AI 厂商已提供稳定的 API（Gemini 2.x、GPT-4o、Claude 3.5、DeepSeek V3），SKILL.md 微型 Agent 架构已在 v0.4.4 中验证可行。

2) Forsion 生态基础已建立：后端统一服务（forsion-backend-service）已完成用户认证、积分计费、多项目模型配置等基础能力，具备跨应用数据整合的技术条件。

3) 用户需求明确：v0.4.x 实验版本的使用反馈表明，多模型切换和个性化主题是用户高频使用的功能，Skills 扩展能力是最期待的新方向。

二、需求目标

目标类型描述衡量指标目标值
核心业务目标 提供统一的多模型 AI 对话入口，让用户无需在多个平台间切换 用户首选使用率（将 AI Studio 作为主要 AI 工具的用户比例） ≥ 60%
差异化目标 通过 Skills 体系赋予 AI "行动能力"，超越纯对话 Skills 安装量 & 使用频次（每用户平均启用 Skills 数） ≥ 3 个/用户
生态目标 通过跨应用数据中枢打通 Forsion 生态，形成用户粘性 跨应用 Skill 调用率（涉及其他 App 数据的对话比例） ≥ 20%
用户体验目标 非技术用户 5 分钟内完成首次对话 + Skill 启用 新用户 onboarding 完成率（从注册到首次成功使用 Skill） ≥ 80%
技术目标 支持私有化部署，用户可使用自有 API Key Docker 一键部署成功率 ≥ 95%

三、用户与使用场景

3.1 典型用户画像

小美 — 内容创作者
  • 28 岁，自媒体运营
  • 每天需要写文章、翻译、做社交媒体文案
  • 不懂技术，希望 AI 工具"开箱即用"
  • 核心诉求：一个地方搞定所有 AI 需求，不想来回切换平台
阿杰 — 小型团队负责人
  • 35 岁，创业公司 CEO
  • 需要团队统一使用 AI，控制成本
  • 关心数据安全，希望自部署
  • 核心诉求：统一管理团队 AI 使用，积分计费透明可控
老陈 — Forsion 生态深度用户
  • 42 岁，项目经理
  • 同时使用 Forsion Desktop 和 Calendar
  • 希望 AI 能理解自己的工作全貌
  • 核心诉求：AI 能自动关联日历日程，在聊天中提醒和辅助决策

3.2 核心用户旅程图

阶段 用户触点 用户行为 痛点 / 情绪 机会点 / 功能转化
1. 发现 官网 / 推荐链接 了解产品，决定试用 不确定是否又是一个普通 AI 工具 首页突出"多模型 + Skills + 跨应用"差异化价值
2. 注册 登录/注册页 使用邀请码注册账号 注册流程繁琐则直接放弃 极简注册（用户名 + 密码 + 邀请码），获赠初始积分
3. 首次对话 主聊天界面 选择模型，发送第一条消息 不知道选哪个模型好，怕选错浪费积分 默认推荐最佳模型，首次对话 0 积分消耗（引导体验）
4. 探索 Skills Settings > Skills 市场 浏览并安装感兴趣的 Skills 不理解 Skill 是什么，概念模糊 每个 Skill 卡片配有一句话描述 + 使用演示，安装即用
5. 深度使用 聊天中调用 Skill 让 AI 联网搜索 / 读取日历 / 执行代码 Skill 失败时不知道原因 Skill 执行过程可视化，失败时给出清晰原因和重试引导
6. 跨应用联动 对话中涉及其他 App 数据 问 AI "明天有什么安排" → AI 读取 Calendar 数据 数据授权流程需要清晰，用户担心隐私 首次跨应用访问弹出授权确认，一次授权永久有效（可撤销）
7. 持续留存 日常工作流 将 AI Studio 作为日常工具，管理多个会话 对话历史管理混乱，找不到之前的重要对话 会话分类 + Emoji 标签 + 搜索 + 置顶功能

四、需求功能清单

4.1 对话核心模块

功能点描述优先级状态
多模型对话 支持 Gemini、GPT、Claude、DeepSeek 等模型，用户可在对话中随时切换模型 P0 已有
流式输出 SSE 实时逐字显示 AI 回复，类 ChatGPT 体验 P0 已有
思考过程折叠 AI 深度思考过程可折叠显示，一键展开查看推理链 P1 已有
富文本渲染 Markdown、LaTeX/KaTeX 数学公式、代码高亮（带语言标签 + 一键复制） P0 已有
文件上传处理 支持图片、PDF、Word 文档上传，AI 可读取和分析文件内容 P0 已有
多会话管理 创建/切换/删除会话，支持 Emoji 标签、会话搜索 P0 已有
对话历史云同步 会话数据从纯 localStorage 迁移至后端数据库，支持跨设备同步 P1 新增

4.2 Skills 体系（核心差异化）

功能点描述优先级状态
SKILL.md 标准格式 每个 Skill 由 YAML frontmatter（工具定义）+ Markdown body（Agent 提示词）组成，统一 Tools 和 Skills 概念 P0 已有
官方内置 Skills 预装核心 Skills：
  • Web Access — 联网搜索 + URL 抓取
  • Code Execution — 代码运行沙盒
  • Productivity — 文件处理、格式转换
  • Workspace — 会话上下文管理（自动注入）
 P0 已有
Skills 市场页面 Settings 中新增 Skills 市场 Tab，展示所有可用 Skills 的卡片列表（图标 + 名称 + 一句话描述 + 安装/卸载按钮） P0 新增
Skill 作用域控制 用户可设置每个 Skill 的启用范围：全局生效（所有 Session 默认启用）或仅在指定 Session 中启用 P0 新增
官方 Skills 文件夹管理 官方 Skills 以 .skill.md 文件形式存放在项目 client/skills/ 目录，构建时打包 P0 已有
自定义 Skill 创建 用户可在 Skills 市场中创建自定义 Skill（填写名称、描述、工具配置、提示词），保存到 Session 级 IndexedDB P1 已有
Skill 执行可视化 对话中 Skill 被调用时，在消息流中显示执行状态（调用中 → 成功/失败），可展开查看详细执行过程 P1 新增
非官方 Skills 渠道（预留） 预留第三方 Skill 分发接口和安装机制，v1.0 不实现，v1.x 规划 P2 预留

4.3 跨应用数据中枢

功能点描述优先级状态
用户级数据分区 后端按用户分区存储数据，每个用户拥有独立的数据空间；各 Forsion App 的数据按 App ID 归类存储在用户分区下 P0 新增
统一用户画像 后端微型 Agent 维护每个用户的统一画像（偏好、使用习惯、常用关键词），所有 Forsion App 共享同一份画像 P1 新增
跨应用 Skill 调用 通过特定 Skills（如 Calendar Skill）访问其他 App 数据。实现方式：Skill 定义 Tools 接口 + 提示词，调用后端数据中枢 API 读写对应 App 数据 P0 新增
数据授权管理 用户首次跨应用访问时弹出授权确认弹窗，说明将访问哪个 App 的什么数据；用户可在 Settings 中查看和撤销已授权的 App P0 新增
App 数据同步接口 各 Forsion App 通过标准 API 将结构化数据推送到后端数据中枢（如 Calendar 推送日程、Desktop 推送文件索引） P1 新增

4.4 用户体验与界面

功能点描述优先级状态
多主题风格 五种主题：Cyber Tech（渐变现代感）、Notion Style（极简单色）、Monet 2.0（清透水彩莫奈风 + 苹果玻璃质感）、Apple Glass（简约玻璃质感多彩风）、Forsion Theme 1（莫兰迪低饱和 + 苹果玻璃质感） + 亮/暗模式 P0 升级
个人资料系统 自定义昵称 + 头像上传（最大 2MB），昵称优先显示 P1 已有
响应式布局 完美适配桌面端和移动端，侧边栏自动收起 P0 已有
新用户引导流程 首次登录后展示 3 步引导：选择模型 → 发送首条消息 → 探索 Skills。非侵入式，可跳过 P1 新增
智能模型推荐 根据用户消息内容自动推荐最合适的模型（如代码问题推荐 DeepSeek，创意写作推荐 Claude），用户可选择接受或忽略 P2 新增

4.5 后台管理与运营

功能点描述优先级状态
用户管理 创建、编辑、禁用用户；角色分配（管理员/普通用户） P0 已有
模型管理 添加自定义 AI 模型、配置 API Key 和 Base URL；项目级模型配置（控制每个 App 可用的模型） P0 已有
积分管理 基于 Token 用量的积分计费、邀请码管理、积分余额和交易记录查看 P0 已有
使用统计 API 调用统计（按模型/日期/用户/项目来源聚合），支持 X-Project-Source 区分来源 P1 已有
官方 Skills 管理 管理员可在后台上架/下架官方 Skills，设置 Skill 的可见性和推荐排序 P1 新增

六、业务流程图

完整的 Mermaid 流程图已产出，存放在 flowcharts/v1.0-core-flows.md，包含 6 个核心流程：

  1. 用户主流程 — 从打开应用到日常使用的整体旅程（登录 → 引导 → 对话 → Skills → Settings）
  2. Skills 安装与作用域管理 — 浏览市场 → 筛选 → 安装 → 配置全局/Session/禁用作用域
  3. Skill 执行可视化 — 消息触发 → 意图识别 → 检查启用/授权 → Tool 调用 → 成功/失败/超时处理
  4. 跨应用数据授权（时序图） — 用户 ↔ AI ↔ 前端 ↔ 后端 ↔ 目标 App 的完整授权与数据获取交互
  5. 新用户引导（状态图） — 3 步引导的状态转换：选择模型 → 发送消息 → 探索 Skills，支持前进/后退/跳过
  6. 积分消耗与模型调用 — 积分检查 → Skills 注入 → API 调用 → Token 消耗计算 → 异常处理

文件位置：flowcharts/v1.0-core-flows.md — 使用任意支持 Mermaid 渲染的 Markdown 查看器打开即可查看流程图

七、异常与边界处理

异常场景系统处理 / 提示文案
网络断开"网络连接已断开，您的消息已保存为草稿，恢复网络后可继续发送。"
API Key 失效"当前模型的 API 配置已过期，请联系管理员更新或切换其他模型。"
积分不足"积分余额不足，无法完成本次对话。" + [查看余额] + [联系管理员] 按钮
模型响应超时30s 无响应时显示"模型响应较慢，请稍候…" + [重试] + [切换模型] 按钮
Skill 执行失败在消息流中标记 Skill 名称 + 失败原因（如"Web Access 失败：目标网站不可访问"），并提供 [重试] 按钮
跨应用数据授权被拒AI 回复"我需要访问你的 [Calendar] 数据来回答这个问题，但尚未获得授权。" + [去授权] 按钮
会话数据为空首屏居中展示欢迎语 + 3 个快捷 Prompt 建议卡片（如"帮我写一封邮件""总结这篇文章""今天有什么安排"）
文件上传失败"文件上传失败（超过 2MB / 格式不支持），请检查文件大小和格式后重试。"

更多异常场景将在原型审查阶段补充。

九、未来演进规划 (Roadmap)

v1.0 当前 MVP — 智能对话 + Skills 市场 + 跨应用数据基础

稳定多模型对话 → 完善 Skills 市场和作用域控制 → 建立用户级数据分区和跨应用 Skill 调用框架 → 首批跨应用 Skill（Calendar）。

v1.5 深度 Agent — CLI 接入 + 统一画像

接入本地/云端 CLI，让 Agent 能执行系统命令 → 后端微型 Agent 自动构建和维护用户统一画像 → 更多跨应用 Skills（Desktop 文件管理、笔记搜索）。

v2.0 开放平台 — 第三方 Skills 生态

开放非官方 Skill 分发渠道 → Skill 开发者工具和文档 → Skills 审核与评分体系 → 多租户企业版。
